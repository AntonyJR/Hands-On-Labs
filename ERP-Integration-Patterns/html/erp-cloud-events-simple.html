Oracle Integration - ERP Cloud Events Lab

Background

Oracle Integration provides native connectivity to Oracle and non-Oracle Software as a Service (SaaS) and On-premises applications, such as Oracle ERP Cloud, Oracle Service Cloud, HCM Cloud, Salesforce.com, Workday, EBS, SAP, NetSuite and so on. Oracle Integration adapters simplify connectivity by handling the underlying complexities of connecting to applications using industry-wide best practices. You only need to create a connection that provides minimal connectivity information for each system.

Use Case

This use case explores the use of Oracle Integration to subscribe to Oracle ERP Cloud Events and push the relevant event information to downstream systems. As part of the lab you would be building the following use case scenario:

  • A user creates a Purchase Order (PO) in ERP Cloud and a PO event is triggered.
  • Oracle Integration subscribes to the PO event, and transforms and pushes the relevant information to a custom table in Oracle Visual Builder Cloud Service.

Create the Integration

  1. On the Home page, click Integrations.
  2. On the Integrations page, click Create.
  3. In the Select an Integration Style dialog, select App Driven Orchestration.
  4. In the Create New Integration dialog, configure the integration:

    1. What do you want to call your integration?

      Enter ERP PO Event VBCS

    2. Identifier: leave the default value (this is the name of the integration in uppercase).

    3. Version: leave the default value.

      (Optional) To change the version number, use the format ##.##.####.

    4. What does this integration do?

      Enter the following text:

      This integration shows you how to use Oracle Integration ERP Cloud Adapter Event with the REST adapter to store a purchase order record in the VBCS PO table.

    5. Which package does this integration belong to?: leave blank.
  5. Click Create.

Create the Trigger

  1. Hover over START and click .
  2. Type ERP in the Search field to find your rest connection.

  3. Select the connection named ERP Conn_zjie.

    The Configure Oracle ERP Cloud Endpoint wizard appears.

  4. Name your endpoint POEVENT.
  5. In the What does this endpoint do? field, enter This endpoint subscribes to the purchase order events.
  6. Click Next.
  7. On the Request page, configure the request properties:
    1. From the Define the purpose of this trigger options, select Receive Business Events raised within ERP Cloud.
    2. In the Business Event For Subscription type Purchase Order.
      3. '

    3. Select the Purchase Order Event.

    4. In the Filter Expr for Purchase Order Event field, enter the following expression:

      <xpathExpr xmlns:ns0="http://xmlns.oracle.com/apps/prc/po/editDocument/purchaseOrderServiceV2/" xmlns:ns2="http://xmlns.oracle.com/apps/prc/po/editDocument/purchaseOrderServiceV2/types/">$eventPayload/ns2:result/ns0:Value/ns0:PurchaseOrderLine/ns0:ItemDescription="Lan Cable "</xpathExpr>

      Enter your user number in the Before You Begin section to get a customized expression.

    5. Click Next.
  8. On the Response page, from the Response Type options, select None.

  9. Click Next.

    A summary of the properties you configured appears.

  10. Review the summary and click Done.

Define the Data Mapping

A Map action named Map to CREATEPO is automatically created. We'll define this data mapping.

  1. Select the Map to CREATEPO action.

  2. Click .

    The Data Mapping page appears.

  3. Map the following fields from the value element in the Sources section, to the corresponding elements with the same name in the request-wrapper element in the Target section:

    • POHeaderId
    • OrderNumber
    • ProcurementBusinessUId
    • ProcurementBusinessUnit
    • SoldToLegalEntity
    • SupplierId
    • Supplier
  4. Click Validate.
  5. Click Close.
  6. Click Save.

Define the Tracking Fields

  1. On the Integration canvas, click , and then select Tracking.

  2. In the Business Identifiers for Tracking dialog, configure tracking fields to identify your instance when a purchase order is created in ERP Cloud:

    1. In the Source section, select OrderNumber, and then click .
    2. Select POHeaderId, and then click .
    3. Select ItemDescription, and then click .

    The selected fields appear in the Tracking Fields table. The OrderNumber field is marked as Primary.

  3. Click Save.
  4. On the Integration canvas, click Save, and then click Close.

Activate the Integration

  1. On the Integrations page, locate your integration

    The name of your integration is ERP PO EVENT VBCS .

  2. Click .

  3. Select Contribute integration mappings to Oracle Recommends.

    This option sends your mappings to the recommendations engine which uses this information to suggest you mappings for similar integrations.

  4. Select Enable Tracing and Include Payload.

    Typically this options are disabled in production environments, but we'll enable them for the purpose of this lab.

  5. Click Activate.

    The activation will complete in a few seconds. If the activation succeeds a confirmation message appears at the top of the page.

Testing the Integration

  1. Open a browser to sign in to the ERP Cloud using the following sign in information:

    1. Select your ERP Cloud environment in the Before You Begin section to get a customized URL.

    2. ERP Cloud URL:

    3. Select your ERP Cloud environment in the Before You Begin section to get a customized user.

    4. User:
    5. Password: the password will be provided during the training.
  2. Select the Procurement tab.
  3. Click Purchase Orders.
  4. Select the Tasks tab.
  5. Click Create Order.
  6. In the Supplier field, enter Dell Inc.
  7. For the rest of the fields, leave the default value.
  8. Click Create.
  9. On the Purchase Order page, in the Additional Information section, edit the row in the Lines table and set the following values:
    1. Type: select Goods
    2. Description: enter Lan Cable

    5. Price: enter 5.00

  11. Write down the order number.

Verify the Order Was Created

  1. Select the Tasks tab.
  2. Select Manage Order.
  3. Search for you order using the order number you wrote down, and then wait for the Status field to change to Open.
  4. In Oracle Integration, in the Navigation menu, select Monitoring, then select Integrations, and then select Tracking.
  5. Click the Order Number and confirm that the flow was successfully completed. The flow is colored in green to indicate that it ran successfully.
