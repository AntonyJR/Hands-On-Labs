Background

This use case uses Oracle Integration (OIC) and ERP Cloud Import Bulk Data services with File-based Data Integration (FBDI) compliant files. The goal is to import ERP data such as account payable invoices using processes in ERP Cloud.

The typical flow for this use case is:

  1. The user uploads an FBDI-based account payable invoice file to an FTP server.
  2. Oracle Integration imports the account payable invoice file to the ERP Cloud.
  3. Oracle Integration receives callback from the ERP Cloud.
  4. Oracle Integration displays the rejected invoices in a VBCS web application.

This lab will explore the ERP adapter and Visual Builder Cloud Service features, and show you how to perform the following tasks:

  • Integrate your ERP with Oracle Integration.
  • Read files from an FTP server.
  • Synchronize account payable invoices to the ERP Cloud.
  • Receive a callback from the Oracle ERP Cloud.
  • Build a dashboard with Visual Builder Cloud Service to monitor exceptions or import rejections.

Create the integration

  1. On the Oracle Integration Home page, click Integrations.
  2. On the Integrations page, click Create.
  3. Select Scheduled Orchestration.

  4. Enter the following values:

    • Name: FBDI APIINV ERP class_id
    • Identifier: leave the default value.

      The default value is the name of the integration in upper case.

    • Version: leave the default value (01.00.0000).

      To change this value enter a version number in the format ##.##.####

    • Description: This integration demonstrates the use of OIC’s ERP Cloud Adapter with the FTP adapter to retrieve an AP Invoices file, import it to the ERP Cloud, and receive a call back from the ERP Cloud.
    • Package: leave blank.

  5. Click Create.

    The Integration canvass appears.

Get the file from an FTP server

  1. Hover over the outgoing arrow for Schedule and click .
  2. Select the FTP connection (FTP Conn 96 06).
  3. In the Oracle Adapter Endpoint Configuration Wizard name your endpoint downloadInvoiceZIP.
  4. Click Next.
  5. From the Operation list, select Download File.
  6. From the Transfer Mode options, select ASCII.

  7. Enter the following values

    1. Input Directory: /upload/public_ftp/<student_id>/in
    2. File Name: apinvoiceimport.zip

    3. Download Directory: /apinvoicezip

      This is the staging directory. It is created automatically.

    Note: verify that the input directory exists (The FTP information is provided as a text file). If it doesn't exist, create a folder named after your student ID in the folder upload/public_ftp/. Then create the subfolders in and out.

  8. Click Next.

  9. Review the Summary page, and click Done.

    The FTP endpoint and a Mapping action appear in the Integration canvas.

  10. Delete the Mapping action.

    This action isn't needed because we aren't passing parameters between the Schedule action and the downloadInvoiceZIP action.

Unzip the file

  1. Hover over the outgoing arrow for downloadInvoiceZIP and click .
  2. Select Stage File.
  3. In the Configure Stage File Action wizard, name your action unzipFile.
  4. Click Next.
  5. In the Configure Operation page, from the Stage File Operation list, select Unzip File.
  6. Click next to the Zip File Name field.

  7. In the Expression Builder, enter the following expression:

    $downloadInvoiceZIP/nsmpr3:DownloadFileToICSResponse/nsmpr1:DownloadResponse/nsmpr1:ICSFiles/nsmpr2:Properties/nsmpr2:filename

  8. Click next to the Zip File Directory field.

  9. In the Expression Builder, enter the following expression:

    $downloadInvoiceZIP/nsmpr3:DownloadFileToICSResponse/nsmpr1:DownloadResponse/nsmpr1:ICSFiles/nsmpr2:Properties/nsmpr2:filename

  10. Click next to the Directory to Unzip field.

  11. In the Expression Builder, enter the following expression:

    /apinvoice

  12. Click Next.
  13. Review the Summary page, and click Done.
